*   **Key Engineering Challenge**: Eliminating frame jitter and thresholding gesture detection to avoid accidental double triggers while managing high-frame-rate webcam feeds.

#### 2. рҹӨ– Enemy AI
*   **Target Objective**: Model intelligent, reactive, and non-predictable behaviors for antagonist actors in Unity 3D tactical engines.
*   **Architectural Strategy**:
    *   **Decision-Making Engine**: Designed using modular **Behavior Trees** containing custom selector, sequence, and decorator nodes rather than nested State Machines to scale complexity.
    *   **Perception Modeling**: Simulated sight/hearing cones using raycasting grids to dynamically calculate spatial alert scores.
    *   **Dynamic Navigation**: Integrated Unity NavMesh query operations with local A* path adjustment to maneuver around mobile obstacles.

#### 3. рҹ§  Predictive Maintenance (GTAE)
*   **Target Objective**: Perform anomaly detection on multivariate sensor networks to identify pre-fault indicators in critical machinery.
*   **Architectural Features**:
    *   **Correlation Graphs**: Construct dynamic correlation matrices from sensor timeseries data using a thresholded Pearson coefficient:
        $$A_{i,j} = \text{threshold}(|\text{corr}(S_i, S_j)|, \tau)$$
    *   **HG-LG-GTAE Model**: A hybrid architecture consisting of:
        1.  **High-Graph (HG) Branch**: Captures high-frequency, immediate sensor fluctuations and local correlations.
        2.  **Low-Graph (LG) Branch**: Captures long-term structural trends and global sensor relationship paths.
        3.  **Graph Transformer Autoencoder**: Combines these features with multi-head attention layers to calculate sensor reconstruction error scores used to isolate faults.
    *   **Diagnostic Dashboard**: Implemented a responsive Web Dashboard with HTML5 local storage to persist diagnostic diagnostic metrics and graph snapshots.

### вҡҷпёҸ SYSTEM WORKFLOW & MATH METHODOLOGY

#### Spatiotemporal Graph Modeling
In our predictive maintenance model, the graph is formulated as $G = (V, E, W)$, where:
- $V$ is the set of $N$ sensor nodes (e.g. power loads, temperature sensors).
- $E$ is the set of edges representing statistical dependency.
- $W$ contains the edge weights, calculated from historical timeseries data.

The Attention-Based Feature Fusion mechanism in the Graph Transformer layer calculates attention coefficients between nodes $i$ and $j$ as:

$$\alpha_{i,j} = \frac{\exp\left(\text{LeakyReLU}\left(\mathbf{a}^T \left[\mathbf{W}\mathbf{h}_i \| \mathbf{W}\mathbf{h}_j\right]\right)\right)}{\sum_{k \in \mathcal{N}_i} \exp\left(\text{LeakyReLU}\left(\mathbf{a}^T \left[\mathbf{W}\mathbf{h}_i \| \mathbf{W}\mathbf{h}_k\right]\right)\right)}$$

Where:
- $\mathbf{W}$ is a shared linear transformation parameter matrix.
- $\mathbf{a}$ is the weight vector of the attention layer.
- $\mathcal{N}_i$ is the neighborhood of node $i$.
- $\|$ denotes the vector concatenation operation.

This multi-attention weight representation is combined with temporal convolutional networks (TCN) to detect structural drifts indicating hardware anomalies before critical failure thresholds are crossed.
